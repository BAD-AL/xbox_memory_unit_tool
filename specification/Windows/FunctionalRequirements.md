# Windows Functional Requirements: xbmut (C#/.NET)

This document defines the requirements for re-creating the `xbmut` tool in the .NET Framework ecosystem, ensuring full parity with the Dart implementation.

## 1. CLI Parity
The application MUST implement the following command-line interface:

| Command | Arguments | Description |
| :--- | :--- | :--- |
| `ls` | `<image_path> [-s] [-t]` | List titles and saves. `-s` for sizes, `-t` for timestamps. |
| `import` | `<image_path> <zip_path>` | Import an Xbox gamesave ZIP. |
| `export` | `<image_path> <path> [zip]` | Export a game/save or `all`. |
| `rm` | `<image_path> <path>` | Recursively delete a game or save. |
| `format` | `<image_path>` | Create a valid 8MB FATX image. |

## 2. Logical Requirements (FATX)
- **Superblock:** Must validate `FATX` signature at offset `0x00`.
- **Hybrid Paradox:** Must correctly map logical 2KB sectors to physical 16KB clusters (TR-3).
- **Directory Terminators:** Must recognize `0x00` and `0xFF` as EOD (TR-5).
- **Path Normalization:** Must strip `UDATA/` prefixes during import and prepend them during export.
- **Attributes:** Must set the System bit (`0x04`) for all `.xbx` files.
- **Metadata Parsing:** Must parse `TitleMeta.xbx` and `SaveMeta.xbx` using UTF-16 LE with BOM.

## 3. Storage Compatibility
- The implementation MUST support standard `.bin` files (flat files).
- The implementation MUST support raw physical drives (USB sticks) using the Windows device pathing convention (`\.\PhysicalDriveX`).

## 4. Acceptance Criteria
- **Parity Check:** An image generated by the .NET version must pass the `parity_check.dart` utility when compared against an image generated by the Dart version (ignoring Volume ID and Timestamps).
- **Xemu Compatibility:** Images produced must be mountable in the Xemu emulator without errors.
